# Phase 02: Flet UI Skeleton - Plan 01

## Objective
Build the visual widget: frameless, transparent, always-on-top floating window with basic UI elements.

## Context
- Roadmap: `@../../ROADMAP.md` (Phase 02)
- Brief: `@../../BRIEF.md`
- Phase 01 Summary: `@../01-foundation/01-01-SUMMARY.md`

## Tasks

### Task 1: Flet App - Basic Window Setup
- **Type:** Feature
- **Files:**
  - Create `ui_skeleton.py`
- **Action:**
  1. Create Flet app with:
     - `page.window_frameless = True` (no title bar)
     - `page.window_always_on_top = True` (stays on top)
     - `page.window_height = 80`
     - `page.window_width = 300`
     - Transparent/semi-transparent background
  2. Center on screen
  3. Add title "CloudWhisper"
  4. Test: `flet run ui_skeleton.py`
- **Verify:**
  - App launches with frameless window
  - Window stays visible when clicking other apps
  - Can close app without errors
- **Done:** ui_skeleton.py creates floating widget

### Task 2: UI Elements - Status & Progress
- **Type:** Feature
- **Files:**
  - Update `ui_skeleton.py`
- **Action:**
  1. Add centered Text label: "Ready" (default state)
  2. Add ProgressRing (circular spinner, initially hidden)
  3. Add color management:
     - Green = Ready (idle)
     - Red = Recording
     - Yellow = Processing
  4. Simple layout: label on top, progress indicator below
- **Verify:**
  - App displays "Ready" in green
  - ProgressRing doesn't show by default
  - Text and spinner are centered
- **Done:** ui_skeleton.py shows status and progress UI

### Task 3: UI Elements - Settings Button
- **Type:** Feature
- **Files:**
  - Update `ui_skeleton.py`
- **Action:**
  1. Add small gear/settings icon button (top-right corner)
  2. Button click toggles a settings panel (not functional yet, just placeholder)
  3. Settings panel shows placeholder text: "Settings (not yet implemented)"
  4. Click button again to collapse
- **Verify:**
  - Settings button visible and clickable
  - Panel expands/collapses on click
  - No layout breaks when panel is open
- **Done:** ui_skeleton.py has settings button with toggle UI

### Task 4: Polish & Documentation
- **Type:** Documentation
- **Files:**
  - Update `ui_skeleton.py`
- **Action:**
  1. Add docstring and comments to code
  2. Add simple README in phase folder: "How to run ui_skeleton.py"
  3. Document Flet version used (from requirements.txt)
  4. Note any Windows-specific behavior
- **Verify:**
  - Code is readable and documented
  - README explains how to test the UI
- **Done:** ui_skeleton.py is clean and documented

## Verification (Overall)
- All 4 tasks complete
- `ui_skeleton.py` runs without errors
- Window is frameless and transparent
- Window stays on top
- Status label and progress ring visible
- Settings button functional (toggle open/close)
